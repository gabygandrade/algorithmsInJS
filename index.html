    Recursion

      Learnings

        Process
      • Before start the problem, think about:
        1. Test scenarios - write these out first
        2. Edge cases - ask if I should address these - like if inputs are all well formatted, what if null or undefined is passed in
      • Make sure I "normalize" the input so that I handle differences btw upper & lowercase, punctuation, etc
      • When talking out loud and get stuck - think: what is another way I could think about this? (remember: counterpart hashtable problem)
        Recursion
        Recursive functions work by invoking the same function with a different input until you reach your base case
      1. Write "if" - bc there will always be at least 2 cases (base case & recursive case)
      2. Handle simplest case
        3. Simplest case = base case = no recursive call needed. The condition for which the answer is known
      3. Write recursive call(s)
        • that call should be passed simpler input, or problem should be made 1 step simpler before recrusive call (so we don't overflow the stack)
        • if recursive call returns a value, we can store that value in some variable
      4. Assume recrusive call works. Ask yourself: What does it do? How does it help?
